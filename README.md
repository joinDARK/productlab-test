# Краткое описание архитектуры
## Сервер (`server/`)
Использована стандартная архитектура Nest.js
- `src/websocket/` - модуль для работы с socket.io. Содержит `websocket.gateway.ts` для отправки сообщений в модуле `avito/`;
- `src/avito/` - модуль запуска puppeteer. В сервисе `avito.service.ts` инициализируется `puppeteer`, происходит вход в профиль Avito и устанавливаем Observer на чат для отслеживания сообщений;

## Клиент (`client/`)
- `hooks/` - директория для кастомных React-хуков;
- `shared/` - переиспользуемые компоненты на клиенте;
  - `lib/` - переиспользуемые методы и константы;
  - `ui/` - UI-компоненты;
- `src/` - содержит только корневые React-компоненты `App.tsx`, `main.tsx` и `index.css` - глобальные стили SPA;

# Инструкция по запуску проекта
1. Устанавливаем пакеты.
```bash
$ scripts/init.sh
```
2. Запускаем сервер. После того, как откроется окно с кодом подтверждения, нужно его ввести вручную.
```bash
$ cd server && npm run start:dev
```
3. Открываем тунель для клиента (порт 5173), а после в `vite.config.ts` добавляем хост в поле `allowedHosts`.
4. Запускаем клиент.
```bash
$ cd client && npm run dev
```

Теперь если пользователь Рушан будет писать, то все сообщения будут дублироваться на клиент.
